What advice can you give to new programmers?


Make Things

I really cannot stress this enough, if you want to be a software developer, you need to develop software. Yes, it sounds obvious, but you’d be amazed how many ‘programmers’ just read books and remember algorithms. An employer will need you to build stuff, make sure you actually can.

Don’t make things more difficult than they need to be

If you find Netbeans easier than Vim, use Netbeans. You’ll have plenty of time to learn Vim later if you want to, but for now, concentrate on programming, not the noise around it.

The beginner programming community seems like a huge ego trip for a lot of people, don’t get bogged down in it, just learn.

Don’t do competitive programming

If you want to do it for it’s own sake, go for it, but don’t think it’ll make you a better software developer. It won’t. It might even make you worse.

Break it down

This is the most essential skill a programmer can have. You don’t have one big problem, you have 100 small ones. Take Candy Crush… Can you make that? Probably not. Can you put a single ‘fruit’ on the screen? Maybe… OK, forget ‘fruit’, can you put a red rectangle on the screen… (that will one day, be a fruit.)

If you have a problem that you can’t solve, break it down, keep breaking it down, until you have a problem so small, you can solve it.

Be careful who you listen to

… And yes, that includes me! There is some really crap advice out there, a few of your fellow students might try to ‘advise’ you… They probably don’t have any more experience than you do, but they’ve got the ego… Try to tell the difference, it’s not always that easy.

Don’t get bogged down in languages

As a beginner, it’s easy to see the language as crucial, and it’s not, most of them are basically the same in principle, and it’s only the details that change. Learn to program, and the languages will fade into the background. Think of it like a car, sure the turn signals might be on the other side, maybe it’s faster, maybe it handles differently… It’s still a car.

Be kind to yourself

Programming is hard, you’ll make mistakes, you’ll struggle. That’s fine, everyone does, don’t feel bad about it. Failure is a natural part of learning, and it makes success taste even sweeter!
